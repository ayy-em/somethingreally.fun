Git and VCS for total beginners

Git is a must-have tool in any coder's arsenal. What is it? Well, Git is a free and open source distributed version control system. It's pretty easy to learn and it's one of the things you're expected to be familiar with as a coder.

What is Version Control?

So what is version control? Imagine two developers working on the same mobile app side-by-side. While one of them works on implementing new designs for the in-app screens, the other one is fixing a sign-up bug. How do they make it so that they can both work on the same piece of code without interrupting each other? That's where version control comes into play. Both developers "clone" a repository (basically, a file drawer containing the whole project) and create a separate "branch" (think of it as a detour) to work on. Then, once the feature is fully done and working fine in that specific branch, all of the additions get merged into the "master" branch. That way, the piece of code that might not be functional while still being worked on will not disrupt the workflow for other developers working on the same app. At certain points in the development cycle, all the developers working on the project merge their branches together, updating the "main" branch and testing to see if there are any conflicts in the additional code submitted by their coworkers.

Apart from that, dev teams frequently introduce code reviews whenever someone requests to merge their local branch (say, one specific to a new feature to be added in the next app update). More importantly, you always have the full history of changes introduced and branches created/merged, which makes fixing bugs way easier.

Why Git?

Git has a lot of advantages over competing solutions. For example, being a distributed system means that there is not a single "end-all-be-all" place for the full version history of the project you're working on.
